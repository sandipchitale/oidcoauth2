GET /get

POST /post

Requires WRITE scope authority.

GET /userinfo

Standard OIDC UserInfo on the authorization server. Accessed with the access token (requires the openid scope). Returns sub always; the profile scope adds name/preferred_username.

GET /whoami & /whereami

No scope required — only a token whose aud is bound to this resource server. A zero-scope (least-privilege) token gets in here while /get & /post reject it. (Audience = "may I enter?"; scope = "may I do X?".)
